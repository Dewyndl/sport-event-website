## Стек технологий (рекомендация)

| Компонент | Технология | Почему |
|-----------|-----------|--------|
| Backend | **Python + FastAPI** | Быстрая разработка, async для VK API, легко деплоить |
| Frontend | **Vue 3 + Vite** (или серверный Jinja2 для MVP) | Реактивный UI, быстрый старт |
| БД | **PostgreSQL** | Надёжно, бесплатно, JSON-поля для гибкости |
| VK | **VK Bot API + Callback API** | Уведомления от имени сообщества |
| Авторизация | **Email/телефон** (основная) + **VK ID OAuth** (привязка) | Требование заказчика — не только VK |
| Деплой | **VPS (Timeweb/Beget)** + Docker | Дёшево, контролируемо |

## Архитектура

```
[Браузер] → [Vue SPA / SSR] → [FastAPI Backend] → [PostgreSQL]
                                      ↓
                                [VK Bot API]
                                      ↓
                           [VK ЛС сообщества]

Авторизация:
  Основная:   Email/Телефон + пароль → JWT-сессия
  Опционально: Привязка VK ID → для получения уведомлений
```

## Схема БД

```sql
users
  id, email, phone, password_hash, name, vk_id (nullable),
  role (participant|organizer), created_at

events
  id, organizer_id (FK users), title, sport_type, city, address,
  datetime, critical_date, max_players, is_prepaid, status
  (draft|registration|closed|completed|cancelled), created_at

registrations
  id, event_id (FK), user_id (FK, nullable for guests),
  guest_name, guest_phone, guest_email,
  status (confirmed|cancelled|queue), registered_at, cancelled_at

activity_log
  id, event_id (FK), user_id (FK), action, details (JSON), created_at

notifications
  id, user_id (FK), event_id (FK), channel (vk|email|telegram),
  message, status (sent|failed), sent_at
```

### Ключевые особенности схемы:
- `users.vk_id` — nullable, т.к. регистрация не обязательно через VK
- `registrations` — поддержка гостей (user_id=null, заполнены guest_*)
- `critical_date` — дедлайн, после которого нельзя выписаться без штрафа
- `activity_log` — единый журнал всех действий

## VK-интеграция

### Компоненты:
1. **VK Community Bot** — от имени сообщества отправляет уведомления
2. **Callback API** — endpoint на бэкенде для подтверждения
3. **VK ID OAuth** — опциональная привязка аккаунта для получения уведомлений

### Сценарии уведомлений:
| Событие | Кому | Сообщение |
|---------|------|-----------|
| Игрок записался | Организатору | "Иванов И. записался на Волейбол 05.03 19:00 (10/14)" |
| Игрок записался | Игроку | "Вы записаны на Волейбол 05.03 19:00, адрес: ..." |
| Игрок отписался | Организатору | "Иванов И. отменил запись на Волейбол 05.03" |
| Место освободилось | Следующему в очереди | "Освободилось место на Волейбол 05.03! Вы записаны." |
| Напоминание | Всем участникам | "Через 2 часа: Волейбол, адрес: ..." |
| Игра отменена | Всем участникам | "Игра Волейбол 05.03 отменена организатором" |

### Ограничения VK API:
- Бот пишет только тем, кто **разрешил сообщения** от сообщества
- Пользователи без VK — уведомления им не приходят (fallback на email)
- Лимит: 20 сообщений/сек

## Риски и сложности

| Риск | Уровень | Митигация |
|------|---------|-----------|
| Не все участники привяжут VK | **Высокий** | Обязательный онбординг с инструкцией; fallback на email/SMS |
| Критичная дата / дедлайн отмены | Средний | Чёткая логика: до даты — свободная отмена, после — блокировка |
| Гостевая регистрация без аккаунта | Средний | Гости не получают уведомления, только организатор видит их |
| Предоплата — неясная логика | Средний | Уточнить: ручная отметка или интеграция с платёжкой |
| Срочность | Высокий | MVP за 4-5 дней, итерации далее |
| Нет домена/хостинга | Средний | Заложить в стоимость или уточнить |

## План реализации

### Фаза 1 — MVP (4-5 дней, ~24 часа)
- [ ] Схема БД + миграции
- [ ] Авторизация: регистрация по email/телефону
- [ ] CRUD мероприятий (организатор)
- [ ] Запись / отмена записи (участник)
- [ ] Настройка VK-сообщества и бота
- [ ] VK-уведомления: запись/отмена → организатору и участнику
- [ ] Минимальный UI: список игр, страница события, запись
- [ ] Деплой на VPS

### Фаза 2 — Полный функционал (+1-2 недели, ~16 часов)
- [ ] Привязка VK-аккаунта (OAuth)
- [ ] Ручное добавление участников / гостей
- [ ] Очередь при заполнении лимита
- [ ] Критичная дата (дедлайн выхода)
- [ ] Признак предоплаты + ручная отметка
- [ ] Журнал активности (UI)
- [ ] Напоминания о играх (cron-задача)
- [ ] Фильтры: по спорту, дате, городу
